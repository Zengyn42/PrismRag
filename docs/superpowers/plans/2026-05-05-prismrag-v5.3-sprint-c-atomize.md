**Goal:** Implement three MCP tools (`atomize_scan`, `atomize_propose`, `atomize_apply`) that let an LLM (Jei) atomize long vault documents into stable KNOW-ID knowledge nodes, with full resume support if interrupted mid-apply.

**Architecture:** All implementation logic lives in `prism_rag/ingest/atomize.py`. MCP tool registrations are in `prism_rag/mcp_server/server.py`. CLI group for offline management in `prism_rag/cli_atomize.py` (registered into `prism_rag/cli.py`). Proposal files (intermediate state) live at `data_dir/atomize-proposals/pending/<id>.json`; applied proposals move to `data_dir/atomize-proposals/applied/`. The LLM provides all semantic judgment between phases 1 and 2; PrismRag only does mechanical operations (ID allocation, file writes, document patching, graph update).

**Protocol:** Three-phase, two-checkpoint design. Phase 1 (`atomize_scan`) reads structure and caches `content_snapshot` per section server-side — snapshots are NOT returned to the LLM (saves ~10,000 tokens on a 20-section doc). Phase 2 (`atomize_propose`) receives `scan_id` + LLM claims, looks up snapshots from cache, writes proposal with `doc_sha`. Phase 3 (`atomize_apply`) writes knowledge files (CAS checkpoint 1 per claim: `claim_status="applied"`), patches doc + updates `atomized_nodes` frontmatter via `write_atomic` (tmpfile + os.rename), then moves proposal to applied/. Crash recovery: re-run reads source doc frontmatter; if all applied KNOW-IDs already in `atomized_nodes` → skip doc patch (document self-proof idempotency).

**Dependency:** Sprint B must be complete (`Registry` must exist).

**Tech Stack:** Python 3.11+, NetworkX (existing), FastMCP (existing), typer (existing), `prism_rag.store.registry.Registry` (Sprint B), `prism_rag.ingest.incremental.ingest_file` (existing), `prism_rag.vault_ops` (existing), pytest.

```python
"""atomize_document tool suite — three-phase protocol for document atomization.

Phase 1: atomize_scan    — read document structure; cache content_snapshots server-side
Phase 2: atomize_propose — LLM submits claims; KNOW-IDs assigned; proposal written
Phase 3: atomize_apply   — create knowledge nodes, patch original doc, update graph

One CAS checkpoint in Phase 3:
  Checkpoint 1 (per-claim): claim_status="applied" after writing knowledge file

Document self-proof idempotency: source doc frontmatter `atomized_nodes` is the proof
  that the doc-patch phase completed. If all applied KNOW-IDs are in `atomized_nodes`,
  skip doc patch on resume (write_atomic: tmpfile + os.rename).
Stale guard: doc_sha mismatch at apply time → terminal "stale" state; re-scan required.
"""
from __future__ import annotations

import hashlib
import json
import re
import secrets
import time
import unicodedata
from datetime import datetime, timezone
from pathlib import Path
from typing import Any

import frontmatter as fm


# Server-side scan cache: scan_id -> {expiry, doc_sha, sections: {section_id: snapshot}}
_SCAN_CACHE: dict[str, dict] = {}
_SCAN_TTL = 86400  # 24h


class ScanExpiredError(Exception):
    """scan_id not in cache or expired — caller must re-scan."""


def _parse_sections(content: str) -> list[dict[str, Any]]:
    """Extract markdown heading sections. Returns content_snapshot for internal cache use."""
    lines = content.splitlines()
    sections = []
    i = 0
    idx = 0
    while i < len(lines):
        m = re.match(r'^(#{1,6})\s+(.+)$', lines[i])
        if m:
            level = len(m.group(1))
            heading = m.group(2).strip()
            line_start = i + 1  # 1-indexed
            j = i + 1
            while j < len(lines):
                nm = re.match(r'^(#{1,6})\s+', lines[j])
                if nm and len(nm.group(1)) <= level:
                    break
                j += 1
            line_end = j
            body = "\n".join(lines[i + 1:j]).strip()
            sections.append({
                "section_id": f"s_{idx}",
                "heading": heading,
                "level": level,
                "line_start": line_start,
                "line_end": line_end,
                "char_count": len(body),
                "has_knowledge_refs": bool(re.search(r'\[\[KNOW-\d{6}', body)),
                "first_200_chars": body[:200],
                "content_snapshot": body,  # stored in _SCAN_CACHE, stripped from MCP return
            })
            idx += 1
        i += 1
    return sections


def _slug(title: str) -> str:
    """Convert title to lowercase hyphenated slug for filenames."""
    s = unicodedata.normalize("NFKD", title).encode("ascii", "ignore").decode()
    s = re.sub(r"[^\w\s-]", "", s).strip().lower()
    s = re.sub(r"[\s_]+", "-", s)
    return s[:40]


def _batch_alloc(n: int, registry) -> list[str]:
    """Allocate n KNOW-IDs atomically. Private to atomize_propose_impl."""
    return [registry.alloc_id() for _ in range(n)]


def atomize_scan_impl(file_path: Path, vault_root: Path) -> dict[str, Any]:
    """Read document structure; cache content_snapshots server-side (TTL 24h).

    Returns sections WITHOUT content_snapshot (prevents ~10,000 token waste).
    Each section has section_id (s_0, s_1, ...) for stable LLM reference.
    Returns scan_id for use in atomize_propose call.
    """
    fp = file_path.expanduser().resolve()
    vr = vault_root.expanduser().resolve()
    rel = fp.relative_to(vr)

    post = fm.load(str(fp))
    content = post.content
    frontmatter_data = dict(post.metadata)

    already_atomized: list[str] = frontmatter_data.get("atomized_nodes", [])
    if isinstance(already_atomized, str):
        already_atomized = [already_atomized]

    doc_sha = "sha256:" + hashlib.sha256(content.encode()).hexdigest()

    ts = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H-%M")
    src_slug = _slug(rel.stem)
    scan_id = f"scan-{ts}-{src_slug}-{secrets.token_hex(4)}"

    sections = _parse_sections(content)

    # Store snapshots in server-side cache — NOT returned to LLM
    _SCAN_CACHE[scan_id] = {
        "expiry": time.time() + _SCAN_TTL,
        "doc_sha": doc_sha,
        "sections": {s["section_id"]: s["content_snapshot"] for s in sections},
    }

    # Strip content_snapshot from MCP return
    public_sections = [
        {k: v for k, v in s.items() if k != "content_snapshot"}
        for s in sections
    ]

    return {
        "scan_id": scan_id,
        "path": rel.as_posix(),
        "doc_sha": doc_sha,
        "node_id": rel.with_suffix("").as_posix(),
        "already_atomized": already_atomized,
        "sections": public_sections,
    }
```

```python
def atomize_propose_impl(
    source_path: str,
    namespace: str,
    scan_id: str,
    claims: list[dict[str, Any]],
    *,
    data_dir: Path,
) -> dict[str, Any]:
    """Assign KNOW-IDs to LLM claims and write proposal file.

    Requires scan_id from a prior atomize_scan call (cache TTL: 24h).
    Retrieves content_snapshots and doc_sha from server-side scan cache.
    Deduplicates claims by sha256(body + section_id).
    """
    from prism_rag.store.registry import Registry

    # Look up scan cache — raises ScanExpiredError if missing or expired
    cache_entry = _SCAN_CACHE.get(scan_id)
    if cache_entry is None or time.time() > cache_entry["expiry"]:
        _SCAN_CACHE.pop(scan_id, None)
        raise ScanExpiredError(f"scan_id '{scan_id}' expired or not found — re-scan required")

    doc_sha = cache_entry["doc_sha"]
    snapshot_map: dict[str, str] = cache_entry["sections"]  # section_id -> content_snapshot

    data_dir = Path(data_dir)
    registry = Registry(data_dir / "registry.json")
    pending_dir = data_dir / "atomize-proposals" / "pending"
    pending_dir.mkdir(parents=True, exist_ok=True)

    ts = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H-%M-%S")
    src_slug = _slug(source_path.replace("/", "-").replace(".md", ""))
    proposal_id = f"atomize-{ts}-{src_slug}"
    proposal_path = pending_dir / f"{proposal_id}.json"

    # Claim dedup: sha256(body + section_id) — skip duplicate semantic content
    seen_hashes: set[str] = set()
    deduped: list[dict[str, Any]] = []
    for claim in claims:
        section_id = claim.get("section_id", "")
        claim_hash = hashlib.sha256(
            (claim["body"] + section_id).encode()
        ).hexdigest()[:16]
        if claim_hash in seen_hashes:
            continue
        seen_hashes.add(claim_hash)
        deduped.append(claim)

    # Batch-allocate KNOW-IDs (single registry lock round-trip)
    kids = _batch_alloc(len(deduped), registry)

    enriched_claims: list[dict[str, Any]] = []
    assignments: list[dict[str, str]] = []
    for claim, kid in zip(deduped, kids):
        section_id = claim.get("section_id", "")
        enriched = dict(claim)
        enriched["knowledge_id"] = kid
        enriched["section_id"] = section_id
        enriched["content_snapshot"] = snapshot_map.get(section_id, "")
        enriched["claim_status"] = "pending"
        enriched_claims.append(enriched)
        assignments.append({"title": claim["title"], "knowledge_id": kid})

    proposal = {
        "id": proposal_id,
        "source_path": source_path,
        "namespace": namespace,
        "scan_id": scan_id,
        "created_at": datetime.now(timezone.utc).isoformat(),
        "status": "pending",
        "doc_sha": doc_sha,
        "claims": enriched_claims,
    }

    proposal_path.write_text(json.dumps(proposal, indent=2, ensure_ascii=False))
    # NOTE: In asyncio MCP server context, dedup_index writes must use AtomicFileOp
    # (asyncio.Lock + run_in_executor) to avoid blocking the event loop with fcntl.flock.
    # See design doc §4.6 AtomicFileOp — asyncio 环境并发安全规约.

    return {
        "proposal_id": proposal_id,
        "proposal_path": str(proposal_path),
        "assignments": assignments,
        "total_claims": len(deduped),
    }
```

```python
from __future__ import annotations

import json
import textwrap
from pathlib import Path

import pytest

from prism_rag.ingest.atomize import atomize_apply_impl, atomize_propose_impl, atomize_scan_impl


def _setup_proposal(tmp_path: Path, content: str) -> tuple[Path, str, Path]:
    """Write a vault doc, scan it, propose one claim, return (vault_root, proposal_id, data_dir)."""
    vault = tmp_path / "vault"
    vault.mkdir()
    doc = vault / "design.md"
    doc.write_text(content)

    data_dir = tmp_path / "data"
    data_dir.mkdir()

    # Scan must precede propose (populates scan cache)
    scan_result = atomize_scan_impl(doc, vault_root=vault)
    scan_id = scan_result["scan_id"]
    section_id = scan_result["sections"][0]["section_id"] if scan_result["sections"] else "s_0"

    claim = {
        "title": "Architecture Decision",
        "type": "decision",
        "section_id": section_id,
        "source_section": "Architecture",
        "source_lines": [5, 8],
        "body": "The architecture uses a three-tier model.",
        "relations": [],
        "confidence": "high",
```

```python
def atomize_apply_impl(
    proposal_id: str,
    *,
    vault_root: Path,
    data_dir: Path,
    skip_graph_update: bool = False,
) -> dict[str, Any]:
    """Apply an approved proposal: create knowledge nodes, patch source doc, update graph.

    One CAS checkpoint for crash recovery:
      1. Per-claim: claim_status="applied" written to proposal after each knowledge file

    Document self-proof idempotency: source doc frontmatter `atomized_nodes` field is the
    idempotency proof for the doc-patch phase. If all applied KNOW-IDs are already in
    `atomized_nodes`, skip doc patch on resume.
    Stale guard: if source doc SHA differs from proposal doc_sha, set status="stale" and abort.
    """
    data_dir = Path(data_dir)
    vault_root = Path(vault_root)
    pending_dir = data_dir / "atomize-proposals" / "pending"
    applied_dir = data_dir / "atomize-proposals" / "applied"
    applied_dir.mkdir(parents=True, exist_ok=True)

    proposal_path = pending_dir / f"{proposal_id}.json"
    if not proposal_path.exists():
        applied_path = applied_dir / f"{proposal_id}.json"
        if applied_path.exists():
            return {"error": f"Proposal '{proposal_id}' already applied", "status": "already_applied"}
        raise FileNotFoundError(f"Proposal not found: {proposal_path}")

    proposal = json.loads(proposal_path.read_text())
    source_path = vault_root / proposal["source_path"]
    knowledge_dir = vault_root / "knowledge"
    knowledge_dir.mkdir(parents=True, exist_ok=True)

    # Stale guard: abort if source doc changed since scan
    if source_path.exists() and "doc_sha" in proposal:
        current_content = source_path.read_text()
        current_sha = "sha256:" + hashlib.sha256(current_content.encode()).hexdigest()
        if current_sha != proposal["doc_sha"]:
            proposal["status"] = "stale"
            proposal_path.write_text(json.dumps(proposal, indent=2, ensure_ascii=False))
            return {"error": "Source document changed since scan — re-scan required", "status": "stale"}

    claims: list[dict] = proposal["claims"]

    # Document self-proof: check source doc frontmatter for already-applied KNOW-IDs
    import frontmatter as _fm
    _already_in_doc: set[str] = set()
    if source_path.exists():
        try:
            _post = _fm.load(str(source_path))
            _already_in_doc = set(_post.metadata.get("atomized_nodes", []))
        except Exception:
            pass

    # Crash recovery: if all claims applied but doc not patched, skip Phase 1
    all_claims_applied = all(c.get("claim_status") == "applied" for c in claims)
    created_nodes: list[str] = []

    if not all_claims_applied:
        # Phase 1: write knowledge node files; checkpoint each claim
        for i, claim in enumerate(claims):
            if claim.get("claim_status") == "applied":
                created_nodes.append(claim["knowledge_id"])
                continue

            kid = claim["knowledge_id"]
            filename = f"{kid}-{_slug(claim['title'])}.md"
            node_path = knowledge_dir / filename

            if not node_path.exists():
                fm_data: dict[str, Any] = {
                    "knowledge_id": kid,
                    "title": claim["title"],
                    "type": claim.get("type", "fact"),
                    "status": "active",
                    "created": datetime.now(timezone.utc).date().isoformat(),
                }
                if claim.get("relations"):
                    fm_data["relations"] = claim["relations"]

                node_content = "---\n"
                for k, v in fm_data.items():
                    if isinstance(v, list):
                        node_content += f"{k}:\n" + "".join(f"  - {item}\n" for item in v)
                    else:
                        node_content += f"{k}: {v}\n"
                node_content += f"---\n\n{claim['body']}\n"
                node_path.write_text(node_content)

            # CAS checkpoint 1: mark this claim applied before moving to next
            proposal["claims"][i]["claim_status"] = "applied"
            proposal_path.write_text(json.dumps(proposal, indent=2, ensure_ascii=False))
            created_nodes.append(kid)
    else:
        created_nodes = [c["knowledge_id"] for c in claims]

    # Phase 2: patch source document — skip if all applied KNOW-IDs already in atomized_nodes (document self-proof)
    _all_applied_kids = set(c["knowledge_id"] for c in claims if c.get("claim_status") == "applied")
    _doc_already_patched = _all_applied_kids and _all_applied_kids.issubset(_already_in_doc)
    if not _doc_already_patched and source_path.exists():
        lines = source_path.read_text().splitlines(keepends=True)
        patch_instructions: list[tuple[int, int, str]] = []  # (start_0idx, end_0idx, kid)

        for claim in claims:
            kid = claim["knowledge_id"]
            content_snapshot: str = claim.get("content_snapshot", "")
            source_lines: list[int] = claim.get("source_lines", [])

            located = False

            # SHA fast-path: trust source_lines if section body matches snapshot prefix
            if source_lines and len(source_lines) == 2 and content_snapshot:
                s = max(0, source_lines[0] - 1)
                e = min(len(lines), source_lines[1])
                section_body = "".join(lines[s:e]).strip()
                prefix = content_snapshot[:50]
                if prefix and section_body.startswith(prefix):
                    patch_instructions.append((s, e, kid))
                    located = True

            # str.find slow-path: locate snapshot text in full content
            if not located and content_snapshot:
                full = "".join(lines)
                probe = content_snapshot[:200]
                pos = full.find(probe)
                if pos >= 0:
                    # Map char offset to line index
                    char_count = 0
                    for li, line in enumerate(lines):
                        char_count += len(line)
                        if char_count > pos:
                            patch_instructions.append((li, li + 1, kid))
                            located = True
                            break

            # conflict: cannot locate section — skip patching for this claim

        # Apply patches bottom-up to preserve line indices
        for s, e, kid in sorted(patch_instructions, key=lambda x: x[0], reverse=True):
            lines[s:e] = [f"\n> 详见 [[{kid}]]\n"]
        patched_body = "".join(lines)

        # Phase 3: atomic write — patch + atomized_nodes frontmatter update (combined write_atomic)
        try:
            import tempfile, os
            post = fm.load(str(source_path))
            existing = post.metadata.get("atomized_nodes", [])
            if isinstance(existing, str):
                existing = [existing]
            post.metadata["atomized_nodes"] = sorted(set(existing) | set(created_nodes))
            # Apply line patches to post.content then write atomically
            patched_content = fm.dumps(post)  # simplified — actual impl patches body then dumps
            with tempfile.NamedTemporaryFile(mode='w', dir=source_path.parent, delete=False, suffix='.tmp') as tf:
                tf.write(patched_content)
                tmp_path_str = tf.name
            os.rename(tmp_path_str, str(source_path))
        except Exception:
            pass

    # Phase 4: incremental ingest (synchronous, skip_embed=True — graph immediately available)
    # Embedding deferred to next `prism-rag embed` batch run.
    embedding_status = "skipped"
    if not skip_graph_update:
        try:
            from prism_rag.ingest.incremental import ingest_file
            from prism_rag.config import PrismRagSettings
            settings = PrismRagSettings()
            settings.vault_path = vault_root
            settings.data_dir = data_dir
            for kid in created_nodes:
                kid_files = list(knowledge_dir.glob(f"{kid}-*.md"))
                if kid_files:
                    ingest_file(kid_files[0], settings=settings, skip_embed=True,
                                skip_leiden=True, skip_persist=True)
            ingest_file(source_path, settings=settings, skip_embed=True, skip_leiden=True)
            embedding_status = "pending"
        except Exception:
            pass

    # Phase 5: move proposal to applied/
    applied_path = applied_dir / f"{proposal_id}.json"
    proposal["status"] = "applied"
    proposal["applied_at"] = datetime.now(timezone.utc).isoformat()
    applied_path.write_text(json.dumps(proposal, indent=2, ensure_ascii=False))
    proposal_path.unlink()

    return {
        "created_nodes": created_nodes,
        "patched_document": proposal["source_path"],
        "proposal_id": proposal_id,
        "graph_available": not skip_graph_update,
        "embedding_status": embedding_status,
    }
```

## Performance Bugs Found During v5.2 Validation (2026-05-06)

Discovered during GPU-backed re-ingest of ZenithLoom (1907 nodes) + NimbusVault (43 nodes).

### Perf Bug P1 — Similarity Linker: Pure Python O(n²) vs NumPy

**File:** `prism_rag/ingest/similarity_linker.py`

**Problem:** `_cosine_similarity()` uses pure Python `sum(x*y for x,y in zip(a,b))`. For 1907 nodes × 4096-dim vectors this is ~1.8M comparisons × 4096 multiplications each — ~100× slower than NumPy BLAS.

**Fix:** Replace the per-pair loop with a single NumPy matrix multiply:

```python
import numpy as np

def link_similar_nodes(graph, vectors, settings) -> int:
    node_ids = [nid for nid in vectors if nid in graph...]
    matrix = np.array([vectors[nid] for nid in node_ids], dtype=np.float32)
    # L2-normalize rows → cosine similarity = dot product
    norms = np.linalg.norm(matrix, axis=1, keepdims=True)
    norms[norms == 0] = 1.0
    matrix /= norms
    sim = matrix @ matrix.T   # shape (N, N) — one BLAS call
    # mask diagonal, threshold, top-K per row
    np.fill_diagonal(sim, 0.0)
    for i, nid_a in enumerate(node_ids):
        row = sim[i]
        top_k_idx = np.argpartition(row, -settings.similarity_top_k)[-settings.similarity_top_k:]
        for j in top_k_idx:
            if row[j] >= settings.similarity_threshold:
                _add_edge(graph, node_ids[i], node_ids[j], float(row[j]))
    ...
```

**Expected speedup:** ~100-1000× for large graphs (1907 nodes: seconds instead of minutes).

**Tests to add:** `tests/test_similarity_linker_perf.py`
- Verify result parity between old and new implementation on small matrix
- Verify timing on 500-node synthetic graph is under 5s

### Perf Bug P2 — LanceDB Persist: 1-row-at-a-time Upsert

**File:** `prism_rag/store/embedding_store.py`, called from `prism_rag/ingest/embedder.py`

**Problem:** `persist_embeddings()` calls `store.upsert(node_id, vec)` in a loop. Each `upsert()` internally does `delete()` (one LanceDB query) + `add()` (one write). For 1907 nodes = **3,814 LanceDB operations**. Observed: 20 minutes to persist 1907 × 4096-dim embeddings.

**Root cause in `persist_embeddings`:**
```python
for node_id, vec in vectors.items():
    store.upsert(node_id, vec)   # ← 1907 × (delete + add) = 3814 ops
```

**Fix:** Add `EmbeddingStore.bulk_upsert(records: dict[str, list[float]])` that:
1. Deletes all matching `node_id`s in one `table.delete(f'node_id IN ({ids})')` call
2. Calls `table.add(all_rows)` once with all vectors as a PyArrow batch

```python
def bulk_upsert(self, records: dict[str, list[float]]) -> None:
    if not records:
        return
    node_ids = list(records.keys())
    id_list = ", ".join(f'"{nid}"' for nid in node_ids)
    try:
        self._table.delete(f"node_id IN ({id_list})")
    except Exception:
        pass
    rows = [{"node_id": nid, "embedding": records[nid]} for nid in node_ids]
    self._table.add(rows)
```

Update `persist_embeddings()` to call `store.bulk_upsert(vectors)` instead of the loop.

**Expected speedup:** 3814 ops → 2 ops. Observed 20 min → expected <30s.

**Tests to add:** `tests/test_embedding_store_bulk.py`
- `test_bulk_upsert_all_persisted(tmp_path)`: bulk-insert 100 rows, verify all readable
- `test_bulk_upsert_overwrites_existing(tmp_path)`: insert, bulk-upsert same IDs with new vecs, verify updated
- `test_bulk_upsert_empty_noop(tmp_path)`: empty dict → no error, table unchanged
