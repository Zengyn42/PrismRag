# PrismRag v5.5 — Atomize Semantic Deduplication Design

**Date**: 2026-05-16  
**Status**: Design (pre-implementation)  
**Preceded by**: v5.4 — Atomize 体验精打磨

---

## 1. Background & Problem

v5.4's atomize pipeline (scan → propose → apply) has only ID-layer idempotency:

- **Within a proposal**: `atomize_propose` deduplicates by `knowledge_id` (first wins)
- **Crash recovery**: `atomize_apply` reads `atomized_nodes` frontmatter, skips existing KNOW-IDs

**Gap: semantic duplicates.** Atomizing the same document twice, or atomizing two documents that describe the same concept, produces separate KNOW nodes with highly similar content. The graph marks them with `semantically_similar_to` edges (score ≈ 0.9) but does not merge them.

---

## 2. Solution Overview

**Batch embedding retrieval at propose stage + `similar_existing` soft warning + Jei chunked decision (5 claims/batch) + provenance edges on reuse + snapshot rollback.**

```
atomize_scan
     ↓
atomize_propose
  ├─ batch embed claims (body)
  ├─ batch ANN search vs existing KNOW nodes
  ├─ similar_existing > threshold → appended to claim result
  └─ Jei chunked decision (≤5 claims/batch): reuse / create
     ↓
atomize_apply
  ├─ reuse path: create MENTIONS edge + CONTEXT_REF node
  ├─ create path: normal KNOW file write
  └─ each reuse decision writes dedup_snapshot (rollback-capable)
```

---

## 3. Five Design Decisions

### D1 — Check Timing: propose stage (not post-hoc dashboard)

Post-apply merging is destructive (edge redirection, ID deprecation, cache invalidation). Not creating a duplicate at propose time is zero-cost. Prevention over remediation.

### D2 — Threshold: 0.90 fallback + dynamic calibration via `prism calibrate`

- **Cold start** (<100 knowledge nodes): use fallback 0.90 directly; no self-calibration to avoid cold-start deadlock
- **Calibrated mode**: `prism calibrate` samples labeled pairs (duplicate / distinct), fits optimal threshold
- `calibrated_model` field in config; model change forces re-calibration

```toml
[dedup]
threshold = 0.90
calibrated_model = ""
min_nodes_for_calibration = 100
```

### D3 — Jei Decision: chunked at ≤5 claims per LLM call

- 40-claim parallel judgment risks Lost-in-the-Middle degradation
- 5 claims/batch ≈ 2000 tokens comparison context, well within reliable attention range
- Large documents split into multiple LLM calls; slight latency increase, major precision gain
- Claims with empty `similar_existing` pass through with zero LLM overhead

**Jei receives per-chunk:**
```json
{
  "claim": {
    "knowledge_id": "KNOW-000031",
    "title": "LangGraph Nodes and Edges",
    "body": "LangGraph is a directed-graph LLM orchestration framework..."
  },
  "similar_existing": [
    {
      "id": "KNOW-000008",
      "score": 0.93,
      "title": "PrismRag Embedding Layers",
      "body_preview": "PrismRag supports two levels of multimodal emb..."
    }
  ]
}
```

**Jei responds:**
- `{"action": "reuse", "reuse_id": "KNOW-000008"}` — concept is the same, reuse existing node
- `{"action": "create"}` — distinct concept, proceed with creation

### D4 — Cross-Document Duplicates: CONTEXT_REF isolation

Reuse creates `(source_note)-[:MENTIONS]->(existing_KNOW)` edge.

Cross-document local relations ("in this doc's context, X depends on Y") attach to a lightweight `CONTEXT_REF` intermediate node, not directly to the reused KNOW node's global topology. This prevents super-node inflation.

```
[note:A] -[:MENTIONS]-> [KNOW-000008]
[note:B] -[:MENTIONS]-> [KNOW-000008]
              ↑
    CONTEXT_REF_A (source=note:A, context_note="...")
    CONTEXT_REF_B (source=note:B, context_note="...")
```

`CONTEXT_REF` nodes are excluded from Leiden clustering and community statistics.

### D5 — Human Review: soft warning, auto-create on timeout (conservative)

| Mode | Behavior |
|------|----------|
| Interactive (single doc) | Conflict → show content diff; timeout → auto-create (conservative, don't merge) |
| Bulk | Jei fully automatic; every reuse decision writes timestamped `dedup_snapshot` |

### D6 — Rollback: dedup_snapshot (structured, not plain log)

```json
{
  "decision_id": "uuid",
  "timestamp": "2026-05-16T12:00:00Z",
  "action": "reuse",
  "claim_title": "LangGraph Nodes and Edges",
  "reused_id": "KNOW-000008",
  "source_doc": "実験/atomize-test-doc.md",
  "similarity_score": 0.93,
  "pre_state": { "mentions_edges": [] }
}
```

Rollback: `prism rollback --decision-id <uuid>`

## 5. Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Jei hallucination causes false reuse (bulk mode) | `dedup_snapshot` supports `prism rollback --decision-id` for precise single-decision rollback |
| Threshold invalidation after embedding model change | `calibrated_model` field; model change triggers forced re-calibration |
| Chunked multi-call latency increase | Only `similar_existing`-non-empty claims trigger LLM judgment; conflict-free claims are zero-cost pass-through |
| CONTEXT_REF node proliferation | CONTEXT_REF excluded from Leiden clustering and community stats |

---

## 6. Core Insight

The real value of this design is not deduplication itself — it is the evolution of the knowledge graph from **passive document mapping** to **active concept networking**. Every reuse + MENTIONS edge builds a bridge between documents that ID-level idempotency can never create. Semantic deduplication is the mechanism; cross-document knowledge connectivity is the goal.
