## 1. Overview and Architecture Diagram

### What v7.0 Adds on Top of v5.7

v5.7 already supports federated multi-graph querying, but has the following core pain points:

| Pain Point | v5.7 Current State | v7.0 Solution |
|------|----------|--------------|
| Graph registration requires manual configuration | `PRISM_GRAPHS` env var filled in manually | `registry_cache.json` auto-discovery, auto-registered after ingest |
| Embedding bridge complexity too high | `_build_embedding_bridges`: O(N×M) full vector comparison, load time grows linearly with node count | Three-layer bridge: O(50²) hub representatives, skeleton built once, fine-grained on-demand lazy |
| New nodes not visible after write | write_note only writes to disk, in-memory graph not updated | Hot-patch immediately injects into in-memory graph, BM25/Lance rebuilt asynchronously |
| No global view | No cross-namespace visualization | Meta-graph HTML: namespace super-nodes + dashed bridge edges + portal links |
| No place to store hub/community info | No manifest, no registry | manifest stores source of truth; registry_cache stores derived data (hub_node_ids, version numbers) |

#### Core Logic Pseudocode

```python
def _write_manifest(out_dir, namespace, vault_path, graph, settings):
    embed_label = f"{settings.embed_backend}/{settings.ollama_model}"
    manifest = {
        "schema_version": 1,
        "namespace": namespace,
        "vault_path": str(vault_path.resolve()),
        "data_dir": str(out_dir.resolve()),
        "ingest_time": datetime.now(timezone.utc).isoformat().replace("+00:00", "Z"),
        "node_count": graph.node_count,
        "edge_count": graph.edge_count,
        "embedding_model": embed_label,
        "embedding_dim": settings.embedding_dim,
        "prismrag_version": __version__,
    }
    manifest_path = out_dir / "manifest.json"
    # Atomic write: write to tmp file then os.replace
    tmp = manifest_path.with_suffix(".json.tmp")
    tmp.write_text(json.dumps(manifest, indent=2, ensure_ascii=False), encoding="utf-8")
    os.replace(tmp, manifest_path)

    # Register to registry_cache (failure does not block ingest)
    try:
        _register_manifest(manifest_path, namespace)
    except Exception as exc:
        logger.warning(f"[ingest] registry_cache registration failed (non-fatal): {exc}")

    return manifest_path
```

#### Interface with Existing Code

- Call `_write_manifest()` at the end of each `ingest_*` function in `cli.py`, after `graph.save()`.
- `KnowledgeGraph.node_count` / `edge_count` already exist, use directly.
- `settings.embedding_dim` and `settings.ollama_model` already exist.

### 2.2 registry_cache.py (new file)

**New file**: `prism_rag/store/registry_cache.py`

Responsible for reading and writing the global namespace registry — the data source for serve auto-discovery.

#### Complete Class/Function Signatures

```python
"""Global namespace registry cache.

Stores per-namespace derived metadata (hub nodes, versions) keyed by namespace.
Written with filelock + atomic rename; read is lock-free.

Location: ~/.prismrag/registry_cache.json
"""
from __future__ import annotations

import json
import logging
import os
from datetime import datetime, timezone
from pathlib import Path
from typing import Any

logger = logging.getLogger(__name__)

CACHE_PATH: Path = Path.home() / ".prismrag" / "registry_cache.json"
_SCHEMA_VERSION: int = 1


class RegistryEntry:
    """Typed view of a single registry_cache entry."""

    manifest_path: str           # absolute path string
    last_seen: str               # ISO-8601 UTC timestamp
    hub_node_ids: list[str]      # representative node IDs (with ns:: prefix)
    hub_set_version: int         # hub set version number, incremented on each rebuild
    community_count: int         # number of communities (Leiden result)


def register(
    manifest_path: Path,
    namespace: str,
) -> None:
    """Upsert namespace entry. Idempotent; safe for concurrent calls.

    Uses filelock (portalocker / filelock library) + atomic rename.
    Only updates manifest_path and last_seen; never overwrites hub_* fields
    if they already exist (to avoid clobbering P1 data from P0 call).
    """

def update_hub(
    namespace: str,
    hub_node_ids: list[str],
    hub_set_version: int,
    community_count: int,
) -> None:
    """Write hub representative node IDs and version after Layer-2 rebuild.
    Called by FederatedGraph._rebuild_hubs().
    """

def discover_all(
    cache_path: Path = CACHE_PATH,
) -> list[dict[str, Any]]:
    """Load cache, stat-probe each manifest_path, evict ghosts.

    Returns list of live entries (manifest_path exists on disk).
    Ghost entries (manifest_path missing) are evicted and cache rewritten.
    Stat probe costs < 1ms per entry on local FS.
    """

def get_manifest(namespace: str, cache_path: Path = CACHE_PATH) -> dict | None:
    """Parse and return a namespace's manifest.json. Returns None on error."""

def get_hub_version(namespace: str, cache_path: Path = CACHE_PATH) -> int:
    """Return hub_set_version for namespace, or 0 if not found."""

def _atomic_write(path: Path, data: dict) -> None:
    """Write dict as JSON to path using tmp-file + os.replace pattern."""

def _load_cache(path: Path) -> dict:
    """Load and validate cache JSON. Returns empty schema on missing/corrupt file."""
```

#### Core Implementation Pseudocode

```python
def register(manifest_path, namespace):
    from filelock import FileLock
    lock_path = CACHE_PATH.with_suffix(".json.lock")
    CACHE_PATH.parent.mkdir(parents=True, exist_ok=True)

    with FileLock(str(lock_path), timeout=5):
        cache = _load_cache(CACHE_PATH)
        entries = cache.setdefault("entries", {})
        existing = entries.get(namespace, {})

        entries[namespace] = {
            **existing,                               # preserve existing hub_* fields
            "manifest_path": str(manifest_path.resolve()),
            "last_seen": _utcnow_iso(),
        }
        # For new entries, initialize hub fields
        if "hub_node_ids" not in entries[namespace]:
            entries[namespace].update({
                "hub_node_ids": [],
                "hub_set_version": 0,
                "community_count": 0,
            })
        _atomic_write(CACHE_PATH, cache)


def discover_all(cache_path=CACHE_PATH):
    if not cache_path.exists():
        return []
    cache = _load_cache(cache_path)
    entries = cache.get("entries", {})
    live = {}
    evicted = []

    for ns, entry in entries.items():
        mp = Path(entry.get("manifest_path", ""))
        if mp.exists():                               # stat probe
            live[ns] = entry
        else:
            evicted.append(ns)
            logger.debug(f"[registry_cache] evict ghost: {ns} ({mp})")

    if evicted:
        cache["entries"] = live
        try:
            _atomic_write(cache_path, cache)
        except Exception as exc:
            logger.warning(f"[registry_cache] ghost cleanup write failed: {exc}")

    return [{"namespace": ns, **entry} for ns, entry in live.items()]
```

### 2.4 Three-Layer Bridge (federated.py changes)

**Modified file**: `prism_rag/store/federated.py`

The existing `_build_embedding_bridges` does a full O(N×M) comparison and must be replaced with the three-layer architecture.

#### Layer-2: Hub Representative Node Selection and Skeleton Bridge

```python
import random
import itertools
from functools import lru_cache

def _select_hub_nodes(
    self,
    namespace: str,
    k: int = 30,
    m: int = 20,
) -> list[str]:
    """Select k top-degree + m random nodes as bridge representatives.

    Args:
        namespace: Target namespace key.
        k:         Top-degree nodes count (default 30).
        m:         Random sample supplement count (default 20).

    Returns:
        List of bare node IDs (no namespace prefix). Length <= k + m.

    Notes:
        - Uses degree centrality (O(E)) rather than betweenness (O(VE)).
        - Random supplement ensures low-degree but semantically important
          nodes (e.g., isolated concept nodes) get representation.
    """
    graph = self._graphs.get(namespace)
    if graph is None:
        return []
    degrees = sorted(graph.g.degree(), key=lambda x: -x[1])
    top_k = [n for n, _ in degrees[:k]]
    pool = [n for n, _ in degrees[k:]]
    rand_m = random.sample(pool, min(m, len(pool))) if pool else []
    return top_k + rand_m


def _build_hub_bridges(
    self,
    stores: dict[str, "EmbeddingStore"],
    threshold: float = 0.70,
    top_k: int = 3,
) -> None:
    """Layer-2: build skeleton bridges using hub representative nodes.

    Complexity: O(H_a × H_b) where H = 50 per namespace.
    Maximum: O(50²) = 2500 comparisons per namespace pair.

    For each hub-pair hit (cosine > threshold):
      - Creates community-level bridge edge.
      - community_id="inbox" nodes are skipped (unclustered hotpatch nodes).

    After completion: writes hub_node_ids + hub_set_version to registry_cache.
    """
    if len(self._graphs) < 2 or not stores:
        return

    for ns_a, ns_b in itertools.combinations(sorted(self._graphs.keys()), 2):
        store_a = stores.get(ns_a)
        store_b = stores.get(ns_b)
        if store_a is None or store_b is None:
            continue

        hubs_a = self._select_hub_nodes(ns_a)    # up to 50 nodes
        hubs_b = self._select_hub_nodes(ns_b)    # up to 50 nodes

        for node_id_a in hubs_a:
            # Skip inbox (unclustered) nodes
            node_data_a = self._graphs[ns_a].g.nodes.get(node_id_a, {})
            if node_data_a.get("community_id") == "inbox":
                continue

            vec_a = store_a.get_embedding(node_id_a)
            if vec_a is None:
                continue

            results = store_b.search_subset(vec_a, node_ids=hubs_b, top_k=top_k)
            for node_id_b, distance in results:
                node_data_b = self._graphs[ns_b].g.nodes.get(node_id_b, {})
                if node_data_b.get("community_id") == "inbox":
                    continue

                similarity = 1.0 - (distance / 2.0)
                if similarity < threshold:
                    continue

                self._bridges.append({
                    "source_ns": ns_a,
                    "source_id": node_id_a,
                    "target_ns": ns_b,
                    "target_id": node_id_b,
                    "relation": "hub_similar",
                    "confidence": "INFERRED",
                    "weight": round(similarity, 4),
                    "source_pass": "hub_ann",
                })

    hub_count = sum(1 for b in self._bridges if b["relation"] == "hub_similar")
    logger.info(f"[federated] hub bridges: {hub_count}")
```

#### Updated `build_bridges` Method

```python
def build_bridges(
    self,
    stores: dict[str, "EmbeddingStore"] | None = None,
    bridge_threshold: float = 0.70,
    bridge_top_k: int = 5,
    probe=None,
) -> int:
    """Build all bridge layers. Replaces v5.7 O(N*M) embedding bridge.

    Layer-1: shared_tag + import/dependency  (deterministic, O(nodes))
    Layer-2: hub ANN skeleton                (O(50²) per ns-pair)
    Layer-3: lazy per-query                  (deferred, not called here)
    """
    self._bridges.clear()
    self._unified = None
    if self._single:
        return 0

    # Layer-1: shared tag bridges (logic same as v5.7, retained)
    self._build_tag_bridges()

    # Layer-1b: import/dependency bridges (new)
    self._build_import_bridges()

    # Layer-2: hub ANN skeleton bridge (replaces v5.7 full embedding bridge)
    if stores and len(stores) >= 2:
        self._build_hub_bridges(stores, bridge_threshold, bridge_top_k)

    if probe is not None:
        for bridge in self._bridges:
            probe.on_edge_created(bridge)

    logger.info(f"[federated] built {len(self._bridges)} bridge edges "
                f"({len(self._graphs)} namespaces)")
    return len(self._bridges)
```

### 2.5 Write Operation Hot-Patching (server.py changes)

**Modified file**: `prism_rag/mcp_server/server.py`

After `write_note` / `patch_note` tools complete a successful write, immediately inject the new node into the in-memory graph and place index rebuilding in a background async queue.

#### Hot-Patch Injection Function

```python
import asyncio
from collections import defaultdict

# Rebuild queue: ns → dirty flag
_rebuild_pending: dict[str, bool] = defaultdict(bool)
_rebuild_lock: asyncio.Lock | None = None   # lazy initialization


async def _hot_patch_node(
    namespace: str,
    node_id: str,
    node_data: dict,
) -> None:
    """Immediately inject a new/updated node into the in-memory graph.

    Step 1: Add node to namespace's KnowledgeGraph in-memory.
    Step 2: Mark community_id = "inbox" (unclustered placeholder).
    Step 3: Invalidate unified_view cache.
    Step 4: Schedule async BM25 + Lance rebuild.

    Does NOT wait for rebuild. Returns immediately.
    """
    fg = _federated
    if fg is None:
        return

    graph = fg.get_graph(namespace)
    if graph is None:
        logger.warning(f"[hot_patch] namespace not loaded: {namespace}")
        return

    # Inject into in-memory graph
    graph.g.add_node(node_id, **{**node_data, "community_id": "inbox"})
    fg._unified = None   # invalidate unified view cache

    # Mark as pending rebuild
    _rebuild_pending[namespace] = True

    # Schedule async rebuild (non-blocking)
    asyncio.create_task(_async_rebuild(namespace))
    logger.debug(f"[hot_patch] injected {namespace}::{node_id}")


async def _async_rebuild(namespace: str) -> None:
    """Background: rebuild BM25, Lance, Leiden, hub nodes for namespace.

    Coalesces multiple rapid writes: if rebuild is already running for
    this namespace, the new flag will trigger another pass.

    Guaranteed consistency: new nodes queryable via graph traversal
    immediately (hot_patch); BM25/ANN lag up to 30 minutes.
    """
    global _rebuild_lock
    if _rebuild_lock is None:
        _rebuild_lock = asyncio.Lock()

    async with _rebuild_lock:
        if not _rebuild_pending.get(namespace):
            return
        _rebuild_pending[namespace] = False

        await asyncio.sleep(0)   # yield to event loop

        graph = _federated.get_graph(namespace)
        if graph is None:
            return

        try:
            # 1. rebuild BM25
            idx = BM25Index()
            idx.build(graph)
            _bm25_indices[namespace] = idx

            # 2. rebuild Lance (async-friendly wrapper)
            await _rebuild_lance(namespace, graph)

            # 3. re-run Leiden clustering
            from prism_rag.cluster.leiden import run_leiden
            await asyncio.get_event_loop().run_in_executor(
                None, run_leiden, graph
            )

            # 4. reselect hub nodes, increment hub_set_version
            hubs = _federated._select_hub_nodes(namespace)
            new_ver = _federated._increment_hub_version(namespace)
            from prism_rag.store.registry_cache import update_hub
            community_count = len(set(
                d.get("community_id", "") for _, d in graph.g.nodes(data=True)
                if d.get("community_id") not in ("", "inbox")
            ))
            update_hub(namespace, hubs, new_ver, community_count)

            # 5. rebuild Layer-2 bridges for affected namespace pairs
            _federated.build_bridges(stores=_embedding_stores)

            logger.info(f"[rebuild] completed for {namespace} (hub_ver={new_ver})")
        except Exception as exc:
            logger.error(f"[rebuild] failed for {namespace}: {exc}", exc_info=True)
            _rebuild_pending[namespace] = True   # reset flag, retry next time


async def _scheduled_rebuild() -> None:
    """Periodic 30-minute rebuild sweep for namespaces with pending inbox nodes."""
    while True:
        await asyncio.sleep(1800)
        for ns in (_federated.namespaces if _federated else []):
            graph = _federated.get_graph(ns)
            if graph is None:
                continue
            has_inbox = any(
                d.get("community_id") == "inbox"
                for _, d in graph.g.nodes(data=True)
            )
            if has_inbox:
                _rebuild_pending[ns] = True
                asyncio.create_task(_async_rebuild(ns))
```

## 5. Error Handling and Degradation Strategy

### 5.1 manifest.json Write Failure

**Scenarios**: disk full, insufficient permissions, tmp file write interrupted.

**Strategy**:
- `_write_manifest` catches all exceptions internally, logs WARNING, **does not interrupt ingest**.
- Idempotent: next `ingest` rewrites the file, does not affect the already-generated `graph.json`.
- A namespace without a manifest will not be found by `discover_all()`, but can still be manually configured via `PRISM_GRAPHS` (backward compatible).

### 5.2 registry_cache.json Read Failure

**Scenarios**: JSON corruption, file lock timeout (5 seconds), concurrent write conflict.

**Strategy**:
- `_load_cache` returns an empty schema `{"schema_version": 1, "entries": {}}` on parse failure, no exception raised.
- `register()` FileLock uses `timeout=5`; on timeout, logs WARNING and skips registration (does not block ingest).
- `discover_all()` on failure logs ERROR and returns empty list; serve continues to start but federated graph is empty; MCP tools degrade to single-graph mode (using `PRISM_GRAPHS` value if set).

### 5.3 Hub ANN Build Failure (Layer-2)

**Scenarios**: EmbeddingStore vector dimension mismatch, Lance database corruption.

**Strategy**:
- `_build_hub_bridges` wraps each namespace pair's ANN call in `try/except`; on failure, skips that pair and logs WARNING.
- Degrades to Layer-1 only (shared_tag bridge); federated graph still usable, just with weaker cross-graph semantic associations.
- `list_namespaces()` return includes `"hub_bridge_status": "unavailable"` so the LLM is aware of the state.

### 5.4 Hot-Patch Injection Failure

**Scenarios**: in-memory graph lock, malformed node_id.

**Strategy**:
- `_hot_patch_node` failure does not affect the already-completed physical write (vault file is persisted).
- Logs WARNING; on next `serve` restart, full reload includes the latest `graph.json`.
- Background `_async_rebuild` on failure resets `_rebuild_pending[ns] = True`, to be retried by the 30-minute scheduled task.

### 5.5 registry_cache Ghost Entries

**Scenarios**: vault deleted, data_dir cleared, but registry_cache still has the corresponding entry.

**Strategy**:
- `discover_all()` probes each `manifest_path` with `Path.exists()` (< 1ms).
- Ghost entries are silently cleared and registry_cache is rewritten.
- Log level: `DEBUG`, no WARNING generated (normal operations scenario).

### 5.6 filelock Notes for WSL

- Use the `filelock` library (based on `os.O_EXCL` + polling), not `fcntl.flock` (incomplete WSL implementation).
- Lock file path: `{cache_path}.lock` (same directory as cache).
- timeout=5 seconds prevents deadlocks; on timeout, catch `FileLockTimeout`, log WARNING, and continue.
