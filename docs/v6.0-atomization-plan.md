# PrismRag v6.0 — Pluggable Atomization & Knowledge Benchmark

> Version: v6.0
> Date: 2026-07-18
> Status: Phase 1 complete, Phase 2-4 planned
> Previous: v5.7 (complete) · Next: v7.0 federation meta-graph (designed, not implemented)
> Companion analysis: [graphrag-comparison.md](graphrag-comparison.md)

## Goal

Make knowledge atomization an independent, pluggable, measurable subsystem —
so that when a better splitting method appears (papers, GraphRAG techniques,
fine-tuned models), it plugs in as one subclass without touching the pipeline.
Strategic context: benchmark numbers are the entry ticket for the NotebookLM pitch.

## Phase 1 — Splitter Interface & KNOT Unification (DONE 2026-07-17)

- `prism_rag/ingest/splitters/` package:
  - `Knot` dataclass — **the ONE canonical atomized type** (KNOT, Knowledge
    Ontology Token / 知识元): text, title, ontology_type, source_section_id,
    context_note, method, metadata. `knowledge_id` deliberately excluded
    (KNOW-ID assignment is the pipeline's job at apply time).
  - `Splitter` ABC: `split(section_text, *, doc_context=None) -> list[Knot]`.
  - 5 registered methods: `passthrough`, `sentence`, `paragraph`,
    `fixed_window` (traditional-RAG-chunking control group), `llm`.
  - `LlmSplitter`: versioned atomization prompt (v1: self-contained /
    atomic / faithful / 4-way ontology / context_note decontextualization),
    injectable `llm_fn` backend (Ollama default, Claude CLI smoke-tested).
  - `to_claim_dict()` adapter → existing `atomize_propose_impl`.
- 67 tests green.

## Phase 2 — Knot Lifecycle & GraphRAG Borrowings

1. **`status` field on Knot** (from GraphRAG claims schema):
   `confirmed | suspected | superseded`. Wire to drift-checker: code change →
   linked KNOTs flagged `suspected`.
2. **Gleanings wrapper splitter** (`llm_gleanings`): re-prompt loop
   ("many atoms missed → add them" + Y/N gate, max N rounds) around any
   LLM-based splitter. Enters the benchmark as its own candidate.
3. Optional structured `payload` on Knot (procedure → commands/params;
   fact → triple projection). Text stays canonical; payload is derived.

## Phase 3 — B1 Benchmark: Splitting Quality

- Dataset: Claim2Atom / FActScore-style references.
- Harness: same sections → every registered splitter → score
  (atomicity, self-containedness, faithfulness, coverage).
- Compared: our splitters vs Dense X Propositionizer vs bare prompt.
- Output: reproducible report tied to splitter name + prompt version.
- Note: GraphRAG ships NO benchmark harness — this is differentiation.

## Phase 4 — Community Reports & global_ask (biggest capability gap)

- LLM-written per-community reports over existing Leiden clusters
  (title / summary / rating / cited findings).
- New MCP tool `global_ask`: map-reduce over community reports for
  vault-level thematic questions.
- Later: selective recompute on incremental updates (GraphRAG delta pattern).

## Out of Scope (v6.0)

- Federation meta-graph → renumbered to v7.0.
- DRIFT search, prompt auto-tuning, noun-graph fast path → rejected
  (see graphrag-comparison.md §5).
- Paper survey (Dense X / Molecular Facts / DecMetrics) → scheduled by boss,
  feeds future splitter subclasses.
